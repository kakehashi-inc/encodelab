# アーキテクチャ — EncodeLab

最終更新: 2026-04-29

## 1. プロセス構成

Electron 標準の 3 プロセス構成。

```text
┌─────────────────────────────────────────────────────────────┐
│ main process (Node.js)                                      │
│  - Window 制御 / electron-updater / IPC ハンドラ            │
│  - ファイル I/O (dialog + fs)                               │
│  - MIME 自動判定 (file-type ESM, 動的 import)               │
│  - ハッシュ計算 (Node.js crypto)                            │
└─────────────────────────────────────────────────────────────┘
              ▲           ▲
              │ IPC       │ contextBridge
              ▼           │
┌─────────────────────────────────────────────────────────────┐
│ preload (sandboxed)                                         │
│  - window.encodelab に IPC API を expose                    │
└─────────────────────────────────────────────────────────────┘
              ▲
              │
┌─────────────────────────────────────────────────────────────┐
│ renderer (React + MUI)                                      │
│  - 変換エンジン (CanonicalValue ベース)                     │
│  - 各エンコーダ / デコーダ (自前実装)                       │
│  - QR 生成 (qrcode) / QR 読取 (jsqr)                        │
│  - UI (3 カラム / Pane / 中央コントロール)                  │
│  - Zustand ストア                                           │
└─────────────────────────────────────────────────────────────┘
```

## 3. データフロー

### 3.1 変換実行 (左 → 右の場合)

```text
[左ペイン: text | binary | image]
        │  PaneValue
        ▼
parsePane(inputType, value)
        │  CanonicalValue
        ▼
adapt(canonical, outputDomain)
        │  CanonicalValue (target domain)
        ▼
serializePane(outputType, canonical, ctx)
        │  PaneValue
        ▼
[右ペイン: text | binary | image]
```

`ctx.computeHash` / `ctx.detectMime` は IPC 経由でメインプロセスを呼ぶ非同期関数。

### 3.2 ファイル選択 (バイナリ入力)

```text
[Open ボタン] → window.encodelab.file.open() → IPC → main: dialog.showOpenDialog
                                                       fs.readFile
                                              ← Uint8Array (+ name)
[MIME 判定]   → window.encodelab.mime.detect() → IPC → main: file-type
                                              ← { mime, ext }
[Pane 更新]
```

### 3.3 ハッシュ出力 (canonical を経由しない特例)

```text
[入力ペイン] ─inputBytesForHash()→ Uint8Array
                                         ↓
                          ctx.computeHash() ─IPC→ main: crypto
                                         ↓
                                テキスト (hex / base64)
                                         ↓
                                 [出力ペイン]
```

`runConversion` は出力タイプが hash 系の場合、parse/adapt/serialize の経路をスキップして
`inputBytesForHash` でユーザーが見ているそのままのバイト列 (テキストなら UTF-8 エンコード) を取り出し、
直接ハッシュする。これにより JSON / YAML 入力が「再シリアライズ後」の文字列としてハッシュされる
不具合を回避している。

## 4. 状態管理 (Zustand)

`src/renderer/store/converter-store.ts` で以下を保持。

| 項目 | 説明 |
| --- | --- |
| `direction` | 矢印方向 (`'rtl'` = `→` / `'ltr'` = `←`) |
| `left` / `right` | 各ペインの `{ category, type, value }` |
| `qrOptions` | QR 生成オプション |
| `errorMessage` | 直近の変換エラー (中央エリアに表示) |

派生関数:

- `inputSide(direction)` / `outputSide(direction)` — direction から入出力サイドを導出

## 5. 拡張時のガイドライン

### 5.1 新規タイプを追加する

1. `src/shared/conversion/catalog.ts` の `CATEGORIES` に `TypeDefinition` を追加。`id` は重複しない値にする。
2. `src/renderer/i18n/locales/{ja,en}.ts` の `type.*` にラベルを追加。
3. `src/renderer/conversion/handlers.ts` の `parseTextType` / `serializeTextType` (display=`text` の場合) に分岐を追加。display が `binary` / `image` のものは `parsePane` / `serializePane` 本体を編集。
4. 必要であれば独自エンコーダを `src/renderer/conversion/{encoders,escape,structured,binary,qr}/` 配下に追加。
5. 入出力に意味的な制約があれば `src/shared/conversion/compatibility.ts` を更新。

### 5.2 ハッシュアルゴリズムを追加する

1. `src/shared/types.ts` の `HashAlgorithm` に値を追加。
2. `src/main/services/hash.ts` の `NODE_ALGO` にマッピングを追加。
3. catalog にタイプ追加 + `src/renderer/conversion/handlers.ts` の `computeHashOf` に分岐を追加。
   ハッシュ出力は `engine.runConversion` 側で canonical 経路をスキップし、`computeHashOf` を直接呼ぶ。

### 5.3 メイン側 API を追加する

1. `src/shared/constants.ts` にチャンネル名を追加。
2. `src/main/ipc/*.ts` に handler を実装し `registerIpcHandlers` から呼ぶ。
3. `src/preload/index.ts` の `IPC_CHANNELS` と `api` に追加。
4. `src/shared/ipc.ts` の `IpcApi` 型を更新。
